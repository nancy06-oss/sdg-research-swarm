## 🤖 Agent Profiles & Responsibilities

### Agent 1 — Researcher Agent 🔍
- **Role:** Intelligence Gatherer
- **File:** `agents/researcher_agent.py`
- **Responsibility:** Takes an SDG topic as input and gathers raw data —
  key facts, global statistics, affected populations, and relevant SDG
  goal numbers. Acts as the first line of intelligence collection.
- **Failure Recovery:** If the primary prompt fails, retries with a
  simplified fallback prompt automatically.

### Agent 2 — Synthesizer Agent 🧠
- **Role:** Pattern Analyst
- **File:** `agents/synthesizer_agent.py`
- **Responsibility:** Receives raw research from Agent 1 and extracts
  structured insights — top 3 key findings, root causes, gaps in current
  solutions, specific SDG target numbers, and recommended intervention
  areas.
- **Failure Recovery:** If synthesis fails, uses a shortened version of
  the research output as a fallback input.

### Agent 3 — Report Agent 📝
- **Role:** Policy Writer
- **File:** `agents/report_agent.py`
- **Responsibility:** Takes both research and synthesis outputs and
  produces a complete professional SDG policy report with executive
  summary, problem analysis, recommended actions, and success metrics.
- **Failure Recovery:** Falls back to a brief summary prompt if the full
  report generation fails.

## 🔗 Chain-of-Thought (CoT) Logic

The swarm handles failures through a 3-layer recovery system:

**Layer 1 — Agent-level recovery:**
Every agent wraps its Gemini call in a `try/except` block.
If the primary prompt fails (rate limit, timeout, bad response),
the agent automatically retries with a simpler fallback prompt
without stopping the entire pipeline.

**Layer 2 — Swarm-level quality check:**
After each agent completes, the orchestrator checks if the output
length is above a minimum threshold (50 characters). If the output
is too short or empty, the swarm re-calls that agent with a
rephrased prompt before passing data to the next agent.

**Layer 3 — Graceful degradation:**
If Agent 2 (Synthesizer) fails completely, the swarm passes a
shortened version of Agent 1's research directly to Agent 3
so the pipeline never fully breaks — a partial report is always
generated.

**Example failure flow:**
```
Agent 1 fails (rate limit)
    → catches exception
    → retries with: "Basic SDG facts about {topic}"
    → succeeds with shorter output
    → swarm checks: len(output) > 50 → passes to Agent 2
Agent 2 succeeds normally
Agent 3 succeeds normally
→ Full report generated ✅
```

## ⚙️ Setup Instructions

### Option A — Google Colab (No install needed)

1. Open Google Colab: [colab.research.google.com](https://colab.research.google.com)
2. Click **File → Open notebook → GitHub**
3. Paste your repo URL and open `SDG_Swarm_Demo.ipynb`
4. Click the 🔑 **Secrets** icon (left sidebar)
5. Add secret: `GEMINI_API_KEY` = your key from [aistudio.google.com](https://aistudio.google.com)
6. Run all cells top to bottom
7. Change the `TOPIC` variable to any SDG issue and activate!

### Option B — Google Cloud Shell (Browser terminal)
```bash
# 1. Open shell.cloud.google.com

# 2. Clone the repo
git clone [https://github.com/nancy06-oss/sdg-research-swarm.git]
cd sdg-research-swarm

# 3. Install dependencies
pip install -r requirements.txt

# 4. Set your API key (never saved to files)
export GEMINI_API_KEY="your_key_here"

# 5. Run the swarm
python app/main.py
```

## 🔐 Security & Guardrails

| Measure | Implementation |
|---------|---------------|
| API key protection | Stored in Colab Secrets or env variables only — never in code |
| .gitignore | Blocks `.env`, `__pycache__`, output files from being committed |
| No hardcoded secrets | All sensitive values passed via `os.getenv()` |
| Input validation | Agent outputs checked for minimum length before passing forward |
| try/except on all agents | Prevents single agent failure from crashing the swarm |
| Fallback prompts | Simplified prompts used on retry — avoids infinite loops |

## ⚡ Technical Innovation

Unlike a standard chatbot that answers in a single pass, this swarm:

1. **Separates concerns** — each agent has one specialized job, making
   outputs higher quality than a single prompt trying to do everything

2. **Passes context forward** — Agent 2 receives Agent 1's full output
   as context, and Agent 3 receives both — building richer understanding
   at each stage (true A2A communication)

3. **Self-heals on failure** — no human intervention needed if an agent
   fails; the pipeline recovers and continues automatically

4. **Produces structured artifacts** — not just a chat response but a
   saved, formatted policy document ready for real-world use

5. **Topic-agnostic** — one swarm can research any of the 17 SDGs just
   by changing the input topic string
